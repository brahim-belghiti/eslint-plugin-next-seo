## What V0.5 adds (3 rules)

These three rules go beyond simple metadata-shape checks and touch new file shapes (`sitemap.ts`, client components) and cross-property invariants (`metadataBase` + relative OG images).

| # | Rule | Severity in recommended | What it catches |
|---|------|-------------------------|-----------------|
| 1 | `valid-sitemap-shape` | `error` | `app/sitemap.ts` returns malformed entries (missing `url`, bad `priority`, bad `changeFrequency`) |
| 2 | `metadata-base-required-for-relative-og-images` | `error` | `openGraph.images` has relative URLs but no `metadataBase` is set in the same metadata |
| 3 | `no-metadata-in-client-component` | `error` | File has `"use client"` directive AND a `metadata` or `generateMetadata` export — silently ignored by Next.js |

## Conventions for new rules

Mirror the shape of existing rules. Specifically:

- For metadata-related rules, mirror `src/rules/og-image-in-metadata.ts` and `tests/rules/og-image-in-metadata.test.ts`.
- For file-gated rules (Rule 1, gated to `sitemap.ts`), mirror `src/rules/single-h1-per-page.ts` for how `context.filename` is used.
- For Rule 3, the `"use client"` directive detection is new — see the rule spec below.

General conventions (unchanged from V0.4):

1. **Visitor**: register `Program(program)` and walk from there.
2. **No false positives**: if a value being checked isn't a static literal (or array literal where applicable), the rule MUST stay silent. Variable references, function calls, computed values — all silent.
3. **Reuse `findProperty(object, name)` from `src/utils/metadata.ts`**.
4. **Message IDs**: one per failure mode. Use `data: { ... }` for placeholders.
5. **Tests**: every rule needs ~6 valid + 4 invalid cases, including silence cases.
6. **TypeScript strict** with `noUncheckedIndexedAccess`.
7. **Always run `./node_modules/.bin/vitest run tests/rules/<rule>.test.ts` after writing each rule.** All tests must pass before proposing the commit.
8. **No emojis. No comments unless the why is non-obvious.**
9. **Use Write for new files, Edit for existing.** Don't write a new file when an Edit will do.

## Rule specifications

### Rule 1 — `valid-sitemap-shape`

- **Severity**: `error`.
- **Type**: `"problem"`.
- **File gate**: only run when `context.filename` ends with `/sitemap.ts` or `/sitemap.tsx`. Mirror how `single-h1-per-page` gates on `page.tsx`/`page.jsx`.
- **What it checks**: the `default export` of an `app/sitemap.ts` file.
  1. Find the `ExportDefaultDeclaration`.
  2. If its declaration is a `FunctionDeclaration`, `FunctionExpression`, or `ArrowFunctionExpression` → analyze it. Otherwise (e.g., default export of a variable) → silent.
  3. Find the function's returned `ArrayExpression` — for arrow with expression body, that's `body` itself; for block body, scan top-level statements for a `ReturnStatement` whose `argument` is an `ArrayExpression`. If not found → silent.
  4. For each element of the array:
     - If element is an `ObjectExpression` → analyze.
     - Else (spread, function call, identifier) → silent on that element.
  5. For each `ObjectExpression` entry:
     - **`url` property**: if the entry has no `url` property → report `entryMissingUrl` on the entry node.
     - **`priority` property**: if value is a numeric `Literal` and is `< 0` or `> 1` → report `invalidPriority` on the value node, `data: { value }`.
     - **`changeFrequency` property**: if value is a string `Literal` not in the whitelist → report `invalidChangeFrequency` on the value node, `data: { value }`.
- **`changeFrequency` whitelist** (define inline in the rule file as a `Set<string>`):
  ```
  always, hourly, daily, weekly, monthly, yearly, never
  ```
- **Message IDs**:
  - `entryMissingUrl`: `` "Sitemap entry is missing required `url` property." ``
  - `invalidPriority`: `` "Sitemap entry `priority` `{{value}}` must be a number between 0 and 1." ``
  - `invalidChangeFrequency`: `` "Sitemap entry `changeFrequency` `{{value}}` is not valid. Use `always`, `hourly`, `daily`, `weekly`, `monthly`, `yearly`, or `never`." ``
- **Tests**:
  - Valid (≥6): non-sitemap file (silent regardless of contents), sitemap with sync function declaration returning valid array, sitemap with async function declaration, sitemap with arrow function (expression body) returning array, sitemap with all optional fields (`lastModified`, `priority`, `changeFrequency`) all valid, sitemap with mixed static + spread entries (silent on spread), sitemap with default export of variable (silent — can't analyze), sitemap with no default export at all (silent).
  - Invalid (≥4): entry missing `url`, `priority: 1.5`, `priority: -0.1`, `changeFrequency: "yearli"` (typo).

### Rule 2 — `metadata-base-required-for-relative-og-images`

- **Severity**: `error`.
- **Type**: `"problem"`.
- **What it checks**: in any file that exports `metadata` or `generateMetadata`:
  1. Find the metadata object (use `findMetadataExport`). Bail if `!result || !result.object`.
  2. Find `openGraph` property — must be `ObjectExpression`. Else silent.
  3. Find `images` property within `openGraph` — must be `ArrayExpression`. Else silent.
  4. Scan array elements for relative URL strings:
     - String `Literal` whose value does NOT contain `://` and does NOT start with `//` → relative.
     - `ObjectExpression` with a `url` property whose value is a string `Literal` matching the same condition → relative.
  5. If any relative URL is found AND the metadata object has no `metadataBase` property → report on the **first** relative URL node found.
  6. If `metadataBase` is set to anything (even a variable reference) → silent. We don't validate its value.
- **Message ID**: `relativeOgImageWithoutMetadataBase`
- **Message text**: `` "`openGraph.images` contains a relative URL (`{{url}}`); set `metadataBase` so Next.js can resolve it to an absolute URL." ``
- **Data**: `{ url }` (the offending string value).
- **Tests**:
  - Valid (≥6): no `openGraph`, `openGraph` without `images`, all-absolute URLs (`https://...`), protocol-relative URLs (`//cdn.../og.png`), relative URLs but `metadataBase` is set (any value), `images` is a variable reference (silent), `openGraph` is a variable reference (silent).
  - Invalid (≥3): single relative URL string, relative URL in object form (`{ url: "/og.png" }`), `generateMetadata` variant returning the same shape.

### Rule 3 — `no-metadata-in-client-component`

- **Severity**: `error`.
- **Type**: `"problem"`.
- **What it checks**:
  1. Check whether the file starts with a `"use client"` directive. The directive is a literal `"use client"` (or `'use client'`) at the very top of `Program.body`, before any non-directive statement.
     - In `@typescript-eslint/parser`, this appears as `ExpressionStatement` whose `expression` is a string `Literal` with value `"use client"`. Iterate `program.body` from index 0; only string-literal `ExpressionStatement`s count as directives, and they must come before any non-directive statement (matches the JS directive prologue rule).
  2. If no `"use client"` directive → silent.
  3. Find the metadata export using `findMetadataExport`.
  4. If found → report on `result.exportNode`.
- **Message ID**: `metadataInClientComponent`
- **Message text**: `` "`metadata` exports are ignored in client components. Move metadata to a parent server component or `layout.tsx`." ``
- **Tests**:
  - Valid (≥4): server component with metadata (no `"use client"`), client component with no metadata, client component with other random exports, file where `"use client"` appears mid-file (not in the directive prologue — silent).
  - Invalid (≥3): `"use client"` + `export const metadata`, `"use client"` + `export function generateMetadata`, `"use client"` + `export const generateMetadata = () => ({...})`.
