## Where the project is (after V0.5)

- Currently on `0.5.0`. Published to npm as `eslint-plugin-next-seo`.
- 17 rules shipped across metadata, JSON-LD, JSX content, and sitemap categories.
- All 17 rules are **single-file** ESLint rules. They cannot detect cross-file or filesystem-level concerns (missing `sitemap.ts`, missing `robots.ts`, route-coverage gaps).
- `src/utils/metadata.ts` is the shared metadata-detection utility used by every metadata-related rule.
- Build is `tsup`-driven. Single entry: `src/index.ts` → `dist/index.{js,cjs,d.ts,d.cts}`.

## What V0.6 adds — a companion CLI

V0.6 ships a CLI tool (`next-seo`) inside the same package. It's a separate binary that complements the ESLint plugin by handling **cross-file and filesystem-level checks** that single-file ESLint rules cannot do.

V0.6 CLI scope (3 checks + 1 summary):

| # | Check | Severity | What it catches |
|---|-------|----------|-----------------|
| 1 | `require-sitemap` | `error` | No `app/sitemap.{ts,tsx,xml}` exists |
| 2 | `require-robots` | `error` | No `app/robots.{ts,txt}` exists |
| 3 | `metadata-coverage` | `info` | Reports how many `page.{ts,tsx}` files have a `metadata`/`generateMetadata` export |

The CLI ships the same package: users get it for free when they install `eslint-plugin-next-seo`. It's invoked as `npx next-seo check` (or via the `next-seo` bin in `node_modules/.bin/`).

### What V0.6 explicitly does NOT include

- `no-metadata-on-blocked-route` — needs middleware parsing; defer to V0.7.
- `sitemap-route-mismatch` — comparing sitemap entries against actual routes; defer.
- `--json` output — defer until someone asks.
- `--fix` / scaffolding (e.g., generating a starter `sitemap.ts`) — defer.
- Configuration files — V0.6 has zero config; only a `--dir` flag for non-standard layouts.

## Architecture

```
src/cli/
├── index.ts          # entry; has the shebang; parses argv, runs the pipeline
├── walker.ts         # locate app/, glob files, parse pages
├── types.ts          # Finding, Accumulator, RouteInfo
├── checks/
│   ├── require-sitemap.ts
│   ├── require-robots.ts
│   └── metadata-coverage.ts
└── reporter.ts       # format + print findings, return exit code
```

**Pipeline:**

```
1. resolve app dir       → walker.findAppDir()
2. glob and parse files  → walker.scan() → Accumulator
3. run each check        → checks/*.run(accumulator) → Finding[]
4. format and print      → reporter.print(findings, summary)
5. exit                  → 1 if any error-level findings, else 0
```

**Type contract** (`src/cli/types.ts`):

```ts
export type Severity = "error" | "warn" | "info";

export type Finding = {
  level: Severity;
  check: string;          // e.g. "require-sitemap"
  message: string;
  file?: string;          // optional file path for line-level findings
};

export type RouteInfo = {
  filePath: string;       // absolute path to page.tsx file
  hasMetadata: boolean;   // has `export const metadata` or generateMetadata
};

export type Accumulator = {
  appDir: string;         // absolute path to app/ directory
  routes: RouteInfo[];
  sitemapFile: string | null;   // absolute path or null
  robotsFile: string | null;    // absolute path or null
};
```

**Check function signature** (uniform across all checks):

```ts
export function run(acc: Accumulator): Finding[];
```

This makes checks pure functions, trivially testable.

## Conventions

1. **Reuse existing utilities.** The CLI's page-parsing logic uses `findMetadataExport` from `src/utils/metadata.ts`. Do not duplicate.
2. **No colors / ANSI codes** in V0.6. Plain text. Add later if asked.
3. **Tests in `tests/cli/`** mirroring `src/cli/` structure. Use vitest.
4. **Each check is a pure function** taking an `Accumulator` and returning `Finding[]`. No I/O inside checks.
5. **All I/O happens in `walker.ts`** (glob, fs.readFile, parse).
6. **The entry point (`index.ts`) has `#!/usr/bin/env node` as the very first line.** No leading whitespace, no leading import statements. tsup preserves shebangs when configured.
7. **Use Node's built-in `fs/promises` and `path`.** No `fs-extra` or similar.
8. **Use `fast-glob` for globbing.** It's the de-facto standard and small. Add as a runtime dependency.
9. **Use `@typescript-eslint/parser` directly** for parsing — same parser the plugin uses, already a transitive dep via `@typescript-eslint/utils`. Add as a direct runtime dep too.
10. **Argument parsing**: use Node's built-in `util.parseArgs` (Node 18.18+, available). No `commander` or `yargs`.
11. **Exit codes**: `1` if any `error` finding, otherwise `0`. `info` and `warn` never fail the run.
12. **No emojis. No comments unless the why is non-obvious.**

## Component specifications

### `src/cli/walker.ts`

Three exported functions:

#### `findAppDir(cwd: string, override?: string): Promise<string>`

- If `override` is provided, resolve it relative to `cwd`, verify it's a directory, return it.
- Otherwise: try `<cwd>/app`, then `<cwd>/src/app`. Return the first that exists.
- If neither exists, throw with a helpful message: `"No app/ directory found. Use --dir to specify a custom location."`

#### `findSitemapAndRobots(appDir: string): Promise<{ sitemapFile: string | null; robotsFile: string | null }>`

- Look for `sitemap.{ts,tsx,xml}` directly inside `appDir`.
- Look for `robots.{ts,txt}` directly inside `appDir`.
- Return absolute paths or null.

#### `scanRoutes(appDir: string): Promise<RouteInfo[]>`

- Glob `**/page.{ts,tsx,js,jsx}` inside `appDir`.
- For each match, read the file and parse with `@typescript-eslint/parser` (with `jsx: true`).
- Use `findMetadataExport` from `src/utils/metadata.ts` to determine `hasMetadata`.
- If a parse fails (syntax error in user's source), skip that file and emit a warning to stderr — do NOT crash the whole CLI.
- Return `RouteInfo[]`.

### `src/cli/checks/require-sitemap.ts`

```ts
export function run(acc: Accumulator): Finding[] {
  if (acc.sitemapFile) return [];
  return [{
    level: "error",
    check: "require-sitemap",
    message: "No `sitemap.ts` found. Add `app/sitemap.ts` so search engines can discover all routes.",
  }];
}
```

### `src/cli/checks/require-robots.ts`

```ts
export function run(acc: Accumulator): Finding[] {
  if (acc.robotsFile) return [];
  return [{
    level: "error",
    check: "require-robots",
    message: "No `robots.ts` found. Add `app/robots.ts` to declare crawler directives.",
  }];
}
```

### `src/cli/checks/metadata-coverage.ts`

Returns one `info`-level finding summarizing the coverage. (Not an error/warn — just a report line.)

```ts
export function run(acc: Accumulator): Finding[] {
  const total = acc.routes.length;
  const withMeta = acc.routes.filter((r) => r.hasMetadata).length;
  return [{
    level: "info",
    check: "metadata-coverage",
    message: `${withMeta}/${total} page files export metadata directly.`,
  }];
}
```

### `src/cli/reporter.ts`

#### `print(findings: Finding[]): number`

- Iterate findings, print one line per finding with the format:
  ```
  <symbol> <check>  <message>
  ```
  where symbol is `✖` for error, `⚠` for warn, `•` for info. (These are unicode but not colored. Acceptable.)
- After all findings, print a summary line:
  ```
  Summary: <N> errors, <M> warnings
  ```
- Return `1` if any error, else `0`.

### `src/cli/index.ts`

```ts
#!/usr/bin/env node
import { parseArgs } from "node:util";
import { findAppDir, findSitemapAndRobots, scanRoutes } from "./walker";
import * as requireSitemap from "./checks/require-sitemap";
import * as requireRobots from "./checks/require-robots";
import * as metadataCoverage from "./checks/metadata-coverage";
import { print } from "./reporter";

async function main() {
  const { values, positionals } = parseArgs({
    options: { dir: { type: "string" } },
    allowPositionals: true,
  });

  const command = positionals[0];
  if (command !== "check") {
    console.error("Usage: next-seo check [--dir <path>]");
    process.exit(2);
  }

  const appDir = await findAppDir(process.cwd(), values.dir);
  const { sitemapFile, robotsFile } = await findSitemapAndRobots(appDir);
  const routes = await scanRoutes(appDir);

  const acc = { appDir, routes, sitemapFile, robotsFile };

  const findings = [
    ...requireSitemap.run(acc),
    ...requireRobots.run(acc),
    ...metadataCoverage.run(acc),
  ];

  console.log(`next-seo — scanning ${appDir}`);
  console.log("");
  process.exit(print(findings));
}

main().catch((err) => {
  console.error(err.message);
  process.exit(2);
});
```
